# Code of Conduct

This project is small. Keep the human overhead small too.

## Expected Behavior

- Be direct without being hostile.
- Critique the work, not the person.
- Stay on topic.
- Assume public package standards: no private project names, no client context, no copied internal baggage.
- Explain tradeoffs clearly when you disagree.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Dismissing security reports or bug reports without reading them properly.
- Publishing private information.
- Turning technical disagreement into a status contest.

## Enforcement

The maintainer may edit, reject, lock, or remove issues, pull requests, and comments that make the project harder to maintain.

Serious or repeated problems may lead to a ban from the repository.
